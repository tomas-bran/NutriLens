LA SOLUCIÓN

NutriLens es un
asistente informativo sobre alimentos

Una app web que analiza la etiqueta de un producto desde una foto o PDF, extrae sus datos con IA multimodal, clasifica el riesgo y aptitud aplicando reglas propias, y permite consultar el historial en lenguaje natural con un chat de tipo RAG.

No reemplaza a un profesional de nutrición. Es un asistente que ayuda a entender la información visible en la etiqueta.

DISEÑO TÉCNICO · PIPELINE

El pipeline del análisis · 8 steps observables

01
validate_file
~10 ms · no IA

MIME, tamaño, hash SHA-256, PDF readable.

02
detect_label_kind
~800 ms · Phi-4-mm

"¿es etiqueta?" como filtro previo barato.

03
extract_with_ia
~4 s · Phi-4-mm

Extracción multimodal → JSON estructurado.

04
validate_schema
~3 ms · Zod

1 reintento correctivo si no cumple schema.

05
apply_rules
~1 ms · puro

Aptitud vegano/celíaco/sin lactosa con blacklists.

06
compute_risk
~1 ms · puro

Tabla de verdad sellos × alérgenos.

07
generate_explanation
~1 s · Phi-4-mini

Texto humano + sanitizer + disclaimer.

08
persist
~120 ms · Postgres

Insert + storage + dedupe por fileHash.

🪟 Cada step deja trace: estado, duración y detalles se renderizan en la UI bajo "Pipeline del análisis" (RF-15).

Eval del modelo

Dataset propio: 15 fotos del super + 10 de Open Food Facts + 10 non-food + 15 intents.

Métricas: Precision/Recall/F1 por campo (alérgenos, sellos, riesgo), exact match en categoría y aptitudes, calibración de confidence.

Thresholds duros: safety-críticos requieren threshold demo incluso para MVP.

Loop: correr eval → analizar errores → bump prompt v1→v2 → re-eval → comparar reportes.

Reportes versionados en evals/results/ · audit trail completo.

PROCESO · OBSTÁCULOS

Problemas que encontramos y cómo los resolvimos

!

Azure OpenAI no aprobado a tiempo

Problema: el formulario de acceso a GPT-4o no se resolvió antes del Sprint 1.
Solución: abstracción IaProvider y switch a Phi-4 en Azure AI Foundry (mismo SDK, otro endpoint). Sin cambios de código.

⚠

Azure Policy bloqueando regiones

Problema: la subscription tenía allowed locations limitadas.
Solución: elegimos East US (permitida) y desplegamos los modelos serverless ahí.

i

SQLite no soporta enums de Prisma

Problema: error P1012 con enum Categoria.
Solución: Postgres 16 en Docker desde el día 1. Mismo stack en dev y demo.

🧪

Phi-4 no devuelve json_object nativo

Problema: el SDK no acepta response_format con Phi.
Solución: JSON forzado vía prompt + helper stripJsonFences() y retry correctivo si Zod falla.

RETROSPECTIVA

Qué aprendimos y qué viene

✓ Qué funcionó bien

  • Specs SDD antes que código — cero ambigüedad en PRs.
  • Skill nutrilens-implement-story consistente entre devs.
  • Cierre automático de work items en ADO al mergear PR.
  • MockIaProvider = cero tokens gastados en CI.
  • Coverage threshold duro detectó dos regresiones a tiempo.

↻ Qué cambiaríamos

  • Pedir el acceso a Azure OpenAI más temprano.
  • Comenzar a armar el dataset de eval desde el Sprint 0.
  • Adoptar Conventional Commits desde el primer PR (lo hicimos a los pocos commits).
  • Definir la paleta de marca antes que los wireframes.

→ Próximos pasos

  • Migrar a GPT-4o cuando llegue la aprobación de Azure OpenAI.
  • Deploy productivo a Azure App Service + Database for Postgres.
  • Búsqueda vectorial real para el RAG cuando el historial crezca.
  • Comparador de productos en chat (US-31 · P2).
  • Document Intelligence para PDFs problemáticos.
