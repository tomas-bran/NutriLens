Pipeline observable

9 steps trazables · todos visibles en la UI

01
validate_file
~10 ms · sin IA

MIME, tamaño, hash SHA-256, PDF readable.

02
detect_label_kind
~600 ms · gpt-5.1

Filtro "¿es etiqueta?" antes de gastar tokens.

03
extract_with_ia
~3 s · gpt-5.1

Imagen → JSON estructurado, con caché por hash + promptVersion.

04
validate_schema
~3 ms · Zod

Reintento correctivo si el JSON no cumple.

05
apply_rules
~1 ms · puro

Aptitud vegano · celíaco · sin lactosa.

06
compute_risk
~1 ms · puro

Fórmula determinística sellos × alérgenos.

07
enrich_with_off
~400 ms · opcional

Open Food Facts completa datos faltantes.

08
generate_explanation
~800 ms · gpt-5.1

2-3 oraciones en español + disclaimer.

09
persist
~120 ms · Postgres

Upsert por fileHash + guarda trace.

Cada step deja trace · estado, duración y detalles se ven en la UI bajo "Pipeline del análisis".

Obstáculos

Con qué problemas nos encontramos

Bloqueante

Ningún modelo multimodal nos funcionaba

Probamos GPT-4o (sin aprobación inicial), Phi-3.5 deprecado, después Foundry con Phi-4-multimodal — las llamadas timeouteaban en cuenta Students (KI-02).

La abstracción IaProvider nos rescató: implementamos AzureOpenAIProvider, conseguimos acceso a gpt-5.1 vía Azure OpenAI, cambiamos IA_PROVIDER y listo. Mismos prompts, cero código de negocio tocado.
Infra

Azure Policy bloqueando regiones

La subscription solo permitía algunas regiones para crear recursos.

Elegimos una región permitida (East US) y desplegamos los modelos serverless ahí. Documentado en un ADR.
DB

SQLite no soporta enums de Prisma

Error P1012 al definir enum Categoria en SQLite. Bloqueaba arrancar el ORM.

Postgres 16 en Docker desde el día 1. Mismo stack en dev y demo, docker compose up y listo.
IA

Phi-4 no devuelve JSON nativo

El SDK no acepta response_format: json_object con Phi, así que la salida venía con fences ```json y texto extra.

JSON forzado vía prompt + helper stripJsonFences() + retry correctivo si Zod no valida la primera respuesta.

Retro

Qué nos llevamos

✓Qué funcionó

  • Specs SDD antes que código · cero ambigüedad en PRs.
  • Skill propia · consistencia entre devs e IA.
  • Cierre automático de ADO al mergear PR.
  • MockIaProvider · CI gratis, sin tokens.
  • Coverage threshold duro · detectó regresiones.

↻Qué cambiaríamos

  • Validar viabilidad de los modelos multimodales antes que el resto.
  • Pedir acceso a Azure OpenAI más temprano.
  • Armar el dataset de eval desde el Sprint 0.
  • Definir paleta antes que los wireframes.

→Próximos pasos

  • Migrar a GPT-4o cuando aprueben Azure OpenAI.
  • Deploy productivo a Azure App Service + Postgres.
  • Búsqueda vectorial para el RAG.
  • Comparador de productos en chat (US-31).
  • Document Intelligence para PDFs.
